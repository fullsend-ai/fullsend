# V6 Simulation: Failed Experiment - Killing Bad Ideas Fast

**Purpose:** Prove the Experiment Agent prevents costly mistakes by surfacing hidden costs that gut-feel approaches miss.

---

## What This Tests

**Experiment:** Test "Geekbot" (async daily standup tool) as replacement for synchronous daily standups

**Hypothesis:** Async updates will save meeting time without losing coordination benefits

**Reality:** Tool saves meeting time BUT creates notification fatigue, context-switching costs, and delayed coordination that EXCEED the meeting time saved.

**Critical Question:** Which approach catches this before scaling org-wide?

---

## Two Approaches

### V6-Agent: With Experiment Agent
- Designs experiment with explicit cost/benefit tracking
- Tracks BOTH benefits (meeting time saved) AND costs (notifications, context switching, delays)
- **Result:** Agent surfaces hidden costs, recommends ABANDON
- **Financial impact:** Kills bad idea before it scales → saves Red Hat millions

### V6-NoAgent: Without Agent (Gut-Feel)
- Focuses on obvious benefit (no more meetings!)
- Misses hidden costs (notification overhead, coordination delays)
- **Result:** Team loves it, recommends SCALE
- **Financial impact:** If implemented org-wide → $1.2M-2.4M/year wasted
